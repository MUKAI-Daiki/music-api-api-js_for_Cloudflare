# music-api-js

[MUSICAPI.ORG](https://sunoapi.org/) を利用して音楽、歌詞、インストゥルメンタルトラックを生成するためのシンプルなAPIラッパーを提供するDenoベースのウェブサーバーです。

このサーバーは、バックエンドの音楽生成サービスの非同期・コールバックベースのワークフローを簡略化するミドルウェアとして機能し、フロントエンドアプリケーションへの統合を容易にします。

## 機能

-   **フルソング生成**: 単一のテキストプロンプトから完全な楽曲（歌詞と音楽）を生成します。
-   **カスタム歌詞から音楽生成**: ご自身の歌詞に合わせて音楽を生成します。
-   **インストゥルメンタルトラック**: スタイルとタイトルに基づいてバックグラウンドミュージックを生成します。
-   **歌詞生成**: テーマから自動的に歌詞を生成します。
-   **非同期フロー処理**: APIのタスクベースシステムをシンプルなポーリングメカニズムで管理します。
-   **デモアプリケーション**: すぐに実行できる3つのフロントエンドデモが含まれています。

## ライブデモ

サーバーを起動すると、ブラウザで `http://localhost:7001` にアクセスして以下のデモページを試すことができます:

-   **シンプル生成** (`index.html`): 「80年代J-POP、失恋ソング」といった説明的なプロンプトからフルソングを生成します。
-   **歌詞から音楽生成** (`lyric.html`): まずテーマから歌詞を生成し、生成されたテキストを選択して楽曲を作成します。
-   **インストゥルメンタル** (`inst.html`): タイトルとスタイルの説明（例:「壮大なファンタジーの冒険、オーケストラ」）からインストゥルメンタルトラックを生成します。

## 動作原理

このサーバーは非同期のMUSICAPI.ORGサービスとのやり取りを簡略化します。

1.  **リクエスト**: クライアント（ブラウザ）がこのサーバーに生成リクエストを送信します（例: `/api/fetchGenerateMusicCustom`）。
2.  **タスク作成**: サーバーがリクエストをMUSICAPI.ORGに転送し、直ちに `taskId` が返されます。このIDがクライアントに返送されます。
3.  **ポーリング**: クライアントが `taskId` を使用してサーバーの `/api/fetchTask` エンドポイントを定期的に呼び出し、結果を確認し始めます。
4.  **コールバック**: 音楽生成が完了すると、MUSICAPI.ORGが設定した `CALLBACKBASE` URL（このサーバーの `/api/callBack` エンドポイントを指します）にPOSTリクエストで結果を送信します。
5.  **キャッシュ**: サーバーがコールバックを受信し、結果データを `./cache/` ディレクトリに `taskId` をファイル名としたJSONファイルとして保存します。
6.  **完了**: クライアントの次のポーリングリクエストがキャッシュされたJSONファイルを検出し、最終的な音楽データ（音声URLを含む）を受け取り、処理を完了します。

## 使い始め方

### 前提条件

-   [Deno](https://deno.com/) ランタイムがインストールされていること。
-   [MUSICAPI.ORG](https://sunoapi.org/) のAPIキー。
-   サーバーのパブリックアクセス可能なURL。これはMUSICAPI.ORGのコールバックに必要です。ローカル開発では、[ngrok](https://ngrok.com/) などのトンネリングサービスを使用してローカルサーバーをインターネットに公開できます。

### 設定と起動

1.  **リポジトリのクローン:**
    ```sh
    git clone https://github.com/code4fukui/music-api-js.git
    cd music-api-js
    ```

2.  **環境の設定:**
    サンプルの環境変数ファイルをコピーして編集します。
    ```sh
    cp .env.example .env
    ```
    `.env` ファイルを開き、APIキーと公開コールバックURLを追加します:
    ```env
    APIKEY=your_api_key_from_musicapi_org
    CALLBACKBASE=https://your-publicly-accessible-url.com
    ```
    *注: `CALLBACKBASE` はMUSICAPI.ORGが実行中のサーバーに到達できる公開URLでなければなりません。*

3.  **サーバーの起動:**
    ```sh
    deno serve -A --port=7001 --host="[::]" server.js
    ```
    `-A` フラグはすべての必要な権限（キャッシュ用のファイルシステムアクセス、ネットワークアクセス）を付与します。

4.  **デモの試行:**
    ウェブブラウザを開き、`http://localhost:7001` にアクセスします。

## APIエンドポイント

このサーバーは以下のAPIエンドポイントを提供します。すべてのエンドポイントは、`/api/fetchTask` と組み合わせて最終結果を取得するために使用できる `taskId` を返します。

-   `GET /api/fetchLyrics`
    -   テーマから歌詞を生成します。
    -   **クエリパラメーター:**
        -   `prompt` (文字列): 歌詞のテーマまたはアイデア。

-   `GET /api/fetchGenerateMusicSimple`
    -   単一のプロンプトから自動的に完全な楽曲（歌詞と音楽）を生成します。
    -   **クエリパラメーター:**
        -   `prompt` (文字列): 希望する楽曲の説明（例:「晴れた日についての明るいポップソング」）。
        -   `negativeTags` (文字列、任意): 生成から除外するタグ。

-   `GET /api/fetchGenerateMusicCustom`
    -   提供された歌詞、タイトル、スタイルを使用して楽曲を生成します。
    -   **クエリパラメーター:**
        -   `prompt` (文字列): 楽曲の歌詞。
        -   `style` (文字列): 音楽のスタイル（例:「ヘビーメタル、女性ボーカル」）。
        -   `title` (文字列): 楽曲のタイトル。
        -   `negativeTags` (文字列、任意): 除外するタグ。

-   `GET /api/fetchGenerateMusicCustomInstrumental`
    -   インストゥルメンタルトラックを生成します。
    -   **クエリパラメーター:**
        -   `style` (文字列): インストゥルメンタルの音楽スタイル。
        -   `title` (文字列): トラックのタイトル。
        -   `negativeTags` (文字列、任意): 除外するタグ。

-   `GET /api/fetchTask`
    -   生成タスクのステータスまたは最終結果を取得します。クライアントはこのエンドポイントをポーリングします。
    -   **クエリパラメーター:**
        -   `taskId` (文字列): 確認するタスクのID。

-   `POST /api/callBack`
    -   MUSICAPI.ORGからの結果を受信するための内部エンドポイントです。直接呼び出さないでください。
